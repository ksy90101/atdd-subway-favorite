# 지하철 경로 조회 - API 테스트 / 문서 자동화
 
## 1단계 - 회원관리 기능

### 요구사항
- 회원 정보를 관리하는 기능 구현
- 자신의 정보만 수정 가능하도록 해야하며 로그인이 선행되어야 함
- 토큰의 유효성 검사와 본인 여부를 판단하는 로직 추가
- side case에 대한 예외처리
    - [x] 이메일 형식 검사
    - [x] 비밀번호와 비밀번호 확인 동일 값인지 검사
    - [x] 모든 값에 빈 값인지 검사
    - [ ] 이메일 중복 검사
- 인수 테스트와 단위 테스트 작성
- API 문서를 작성하고 문서화를 위한 테스트 작성
- 페이지 연동

### 기능목록
- 회원가입
- 로그인
- 로그인 후 회원정보 조회/수정/삭제
